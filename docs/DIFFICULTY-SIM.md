# 难度曲线模拟测试 / Difficulty-curve simulation (#43)

用脚本船长在真实游戏里跑完整战役,量化「难度 × 战区」曲线。
Harness 在 `tests/sim/difficulty-sim.js`,原始数据在 `tests/sim/results.json`。

## 怎么跑

```bash
node tests/sim/difficulty-sim.js            # 全矩阵:6 海战战区 × 3 难度 × 10 局(约 40 分钟)
node tests/sim/difficulty-sim.js --pilot    # 冒烟:1 战区 × easy × 5 局
```

不属于 `npm test` / `npm run smoke` 套件,手动跑。需要本机 Chrome(Playwright `channel:'chrome'`)。

## 方法

- headless Chrome 里启动真实游戏,直接驱动 `update(0.05)` 推进时间——一场战争几秒钟墙钟时间跑完。
- 脚本船长的策略(7 轮调参的结果):开局买 5 门炮;**跟着己方舰队走**(舰队屏护、补给舰回血);
  近敌 260m 内转向**绕圈横舷输出**(直冲会被撞击引爆,停船会被集火);海战打赢
  (存活敌舰 ≤2 且已击沉 ≥3)才**压向敌方海港**,port 轨道收紧到 235m 保证 autoGun
  的 260m 对岸轰击射程始终覆盖;全程 `KeyW` 不松——**移动就是装甲**。
- 固定**战列舰**船体(驱逐舰试跑时活不到能产生可比数据)。
- 每局全新 browser context(干净 localStorage),生涯加成不会跨局泄漏。
- 结局四种:win / loss(战败)/ death(首次阵亡)/ timeout(存活到 1500s 上限)。
- 战区 2(The Landing)是纯地面登陆战,步兵 bot 超出 v1 范围,故测的是 6 个海战战区
  (index 0,2,3,4,5,6 = 战区 1,3,4,5,6,7)。

**已知边界(同时也是发现之一):** bot 拥有全部**被动**系统——舰队屏护、补给舰回血、
自动炮塔、额外甲板炮——但用不了两个**主动**玩家技能:手动远程瞄准射击、损管抉择。
所以胜率读数≈0 是**设计使然**;可比的曲线信号在存活时间、击沉数、承伤量里。

## 发现

**1. 玩家操作是决定性的(正面设计证据)。** bot 享有全部被动系统,连 easy 训练湾都赢不了
——胜利必须由手动远程炮击 + 损管这两个玩家技能提供。也就是说游戏的胜负确实押在
玩家「做的事」上,而不是数值堆叠;这是好设计的证据,不是缺陷。

**2. 难度梯度干净、方向正确。** 最可控的对照组是战区 1(双方兵力最少、随机性最小):
存活时间 easy 398s → normal 288s → hard 226s,单调递减,10/10 全部阵亡,
说明 `DIFFICULTY` 乘数在真实战局里确实产生了预期强度差。战区 3 里 easy(703s)
也显著高于 normal/hard(377s / 521s),不过 normal 与 hard 出现交叉——大战区
随机性大,两档的区分要靠 timeout 率和小战区数据一起看(见发现 3)。

**3. easy 的「宽容度」在 timeout 率上清晰可见。** bot 活满 1500s 的局数:
easy 23/60,normal 12/60,hard 14/60。easy 几乎翻倍于另外两档——正是想给
新手的体验(容错时间长);normal 与 hard 的差异则主要体现在小战区的存活时间上,
timeout 率没有再拉开(12 vs 14,样本内属噪声)。

**4. 击沉数随战区规模上升**(T1 ~9-15 → T7 ~21-59):大战区里友军更多、增援更密,
bot 被屏护得更久,自动武器输出时间更长。这与战区推进 = 战斗规模升级的设计一致。

**5. 火灾暴击叠加是 bot 的头号死因(附调参建议)。** T1 诊断局的承伤拆解:
炮弹直伤 207 vs 暴击伤害 216(占了一半还多),暴击构成 {fire: 4, engine: 2},
且 4 座炮塔全部被打瘫。**4 层火同时烧**掉了将近半个血池——对新手这基本是
「看不懂发生了什么就沉了」。

> **建议(作为后续 issue,#43 本身不改玩法):** easy 难度下把玩家船的同时起火数
> 上限设为 2。方向性数据支撑:若 4 层火压到 2 层,该诊断局的暴击伤约砍半
> (火灾占暴击伤的主体),T1 easy 的平均存活有望从 ~400s 显著上移,而 normal/hard
> 曲线不受影响,梯度反而更陡、更符合各档定位。

## 边界与后续

- 战区 2(地面战)未测——需要步兵 bot(v2 候选)。
- bot 不会手动炮击/损管,胜率永远读不出来;若将来想测「可赢性」,
  需要给 bot 加一个简化的手动炮击模型(按命中率打折)。
- 固定战列舰船体;换 `SIM_SHIP=destroyer` 环境变量可跑别的船,但数据不可与本表混比。
- 调难度数值后重跑全矩阵,对比 `tests/sim/results.json` 的历史版本即可量化改动效果。
